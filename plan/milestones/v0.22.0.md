# v0.22.0 — Theme platform foundations

The first slice of the {% ref "SPEC-094" /%} theme-platform epic: make the theme system ready
to support themes beyond Lumina (magazine / editorial / business) — **without yet building
one**. Two independent threads — *tokenize + generate* and *gallery + visual-regression
harness* — plus the *spike* that sizes the v0.23.0 skeleton/skin extraction.

Deferred to later slices: the wholesale skeleton/skin extraction and `data-surface` (v0.23.0),
and theme-owned fonts, new layouts, the theme scaffold, and an actual second theme (later).

## Tokens — tokenize typography, generate the CSS

- {% ref "WORK-404" /%} — extend the token contract with a typographic system (scale, leading, weight, tracking, display family). **The gate.**
- {% ref "WORK-405" /%} — refactor Lumina's ~351 hardcoded type declarations onto the new tokens (no visual change).
- {% ref "WORK-406" /%} — generate Lumina's token CSS from `tokens.ts`; retire the hand-maintained mirror + coverage test.

## Gallery + harness — the safety net (and AI-iteration surface)

- {% ref "WORK-407" /%} — gallery generator, rune subject (every rune × variant, via the HTML adapter, deterministic).
- {% ref "WORK-408" /%} — gallery generator, layout fixtures (the four existing layouts over a synthetic context, multi-viewport).
- {% ref "WORK-409" /%} — shared Playwright visual-regression harness; Lumina baselines; pinned CI container; opt-in package.

## Spike — size the extraction

- {% ref "WORK-410" /%} — skeleton/skin split spike on a small slice; sets the correctness-not-taste cut line, the cascade-layer strategy, the packaging decision, and the v0.23.0 scope estimate.

## Image-src scheme sugar ({% ref "SPEC-106" /%})

Resolve custom schemes in Markdown image `src`s to inline SVG — `placeholder:<shape>`
(deterministic, theme-tinted fixture/draft placeholders) and the `icon:<name>` shorthand.
Surfaced by the gallery work: the first (`data:image/svg+xml`) placeholder attempt was
rejected by the parser, and the scheme is the proper fix.

- {% ref "WORK-418" /%} — image-src scheme registry + resolver hook in the `image` transform.
- {% ref "WORK-419" /%} — `icon:<name>` resolver (the icon-rune inline shorthand).
- {% ref "WORK-420" /%} — `placeholder:<shape>` resolver (token-tinted generated SVG).
- {% ref "WORK-421" /%} — migrate fixtures to `placeholder:` (drop the base64-PNG interim).
- {% ref "WORK-422" /%} — image-scheme authoring docs.

## Sequencing

The two threads run in parallel. Tokens: {% ref "WORK-404" /%} → {% ref "WORK-405" /%} → {% ref "WORK-406" /%}. Harness: {% ref "WORK-407" /%} → {% ref "WORK-408" /%} → {% ref "WORK-409" /%}. The spike ({% ref "WORK-410" /%}) lands late — it benefits from tokenized type ({% ref "WORK-405" /%}) and uses the harness ({% ref "WORK-409" /%}) for its before/after diff — and its output scopes **v0.23.0** (skeleton/skin extraction + `data-surface`).
