# v0.14.0 ‚Äî Design tokens contract + brand identity

The v0.14.0 milestone establishes refrakt's design tokens contract as a typed, theme-agnostic surface ({% ref "SPEC-048" /%}), refreshes the brand mark with a monochrome prism logo ({% ref "SPEC-050" /%}), aligns tint vocabulary with the rest of the token system ({% ref "SPEC-053" /%}), ships Lumina with a neutral default palette plus two opt-in presets ‚Äî `tideline` and `niwaki` ‚Äî that demonstrate the preset architecture ({% ref "SPEC-051" /%}), and lets authors lock or unlock tint modes per page or subtree via frontmatter cascade ({% ref "SPEC-052" /%}).

Together these are the foundational pieces that v1.0 needs in place: a coherent token surface, a brand identity that doesn't compete with the user's, and the preset / cascade mechanics that make Lumina genuinely re-skinnable.

## Implementation order

Specs ship in dependency order rather than numeric order:

1. **{% ref "SPEC-048" /%}** ‚Äî typed token contract, preset merge, dark-mode-as-overlay. Foundation; everything else depends on it.
2. **{% ref "SPEC-050" /%}** (independent, parallel-able) and **{% ref "SPEC-053" /%}** (tint vocabulary cleanup; lands before SPEC-051 ships tint code in the new vocab).
3. **{% ref "SPEC-051" /%}** ‚Äî neutral default palette, syntax + status palettes, tideline preset, niwaki preset, fonts, docs pages.
4. **{% ref "SPEC-052" /%}** ‚Äî per-page and per-subtree tint cascade; refrakt site adopts dark-locked default with auto docs.
