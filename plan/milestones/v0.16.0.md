# v0.16.0 — Registry-driven sites

v0.16.0 turns the entity registry into pages and listings declaratively,
delivers the three sibling registry-query runes
(`collection` / `relationships` / `aggregate` — items / edges / numbers),
and proves the system by scaffolding a real plan site from refrakt's own
`plan/` content. Builds directly on v0.15.0's embedding primitives
(`ref` / `expand`).

## What shipped

### Registry-query runes

- **Collection rune** ({% ref "SPEC-070" /%}) — plural counterpart to
  `ref`/`expand`. Queries the registry (filter / sort / group / limit) and
  projects entities into lists, grids, and tables. Shared field-match
  grammar, deferred-body per-item templates, heading-delimited table
  columns, and formatter functions.
- **Relationships rune** ({% ref "SPEC-072" /%}) — graph-edge counterpart
  to `collection`. Renders an entity's edges grouped by kind, generic over
  any domain's relationship vocabulary.
- **Aggregate rune** ({% ref "SPEC-076" /%}) — number-projecting sibling
  to the above. Single-integer inline form and body-zoned per-group form
  with a `value` sub-filter that drives progress-bar ratios. Forms the
  basis of the plan-site dashboard composition.

### Site machinery

- **Plugin-contributed routes** ({% ref "SPEC-069" /%}) — a
  `contributePages` pipeline phase plus a declarative `entityRoutes`
  config that synthesises one page per registered entity. `embed()`
  embeddability contract for cross-page composition.
- **Plan site scaffolding** ({% ref "SPEC-071" /%}) — refrakt's own plan
  site rebuilt from `plan/` via `entityRoutes` + `collection` /
  `aggregate`. The bespoke `plan build` / `plan serve` commands retired.
  Dashboard composition (aggregate header + per-status `collection`
  panels + empty-state hints) shipped as the canonical scaffold template.

### Chrome and polish

- **Theme toggle rune** ({% ref "SPEC-073" /%}) — light / dark / auto
  toggle as both a chrome slot and a `{% theme-toggle /%}` rune, with
  shared behavior and prod-build CSS parity.
- Accordion polish — leading rotating chevron, native `<details>` slide
  animation via `::details-content` + `interpolate-size`, dividers-only
  outer treatment.
- Badge restyle to a compact sentiment-tinted chip (sentiment via
  `color-mix(in srgb, var(--meta-color) X%, transparent)`).
- Catalog reorganisation — new "Registry" category for the
  cross-page-query runes (`xref` / `expand` / `collection` /
  `relationships` / `aggregate`); seven previously-missing runes added
  (`xref`, `badge`, `gallery`, `showcase`, `bg`, `tint`, `blog`).

### Schema and docs corrections

- `refrakt.config.json` schema — `theme` is now `string | SiteThemeConfig`
  (was just `string`); new `SiteThemeConfig` definition with `package`,
  `presets`, `tokens`, `modes`, `code.colorScheme`. `highlight` flagged
  as legacy in favour of `theme.presets` + `theme.code.colorScheme`.
- `site/content/runes/aggregate.md` — full reference page with live
  previews.
- Three small bug fixes — nav badge wrapping under links, mobile
  breadcrumb overflowing the toolbar, conversation rune's `speakers`
  attribute mis-rendering names as a separate label.
