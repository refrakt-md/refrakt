# v0.14.1 — Syntax token contract extension

The v0.14.1 milestone implements {% ref "SPEC-056" /%} — widens the `SyntaxTokens` contract from 7 required + 2 optional roles to 7 required + 9 optional, with fallback chains via CSS `var()` so simple presets stay simple and richer presets gain the headroom to faithfully carry well-known palettes (Nord, Dracula, Solarized, Catppuccin, Tokyo Night, GitHub, …) without losing the role splits their authors designed in.

Nord is imported as the first non-refrakt syntax preset and serves as the validation case for the contract shape — if Nord's 16 hues land on distinct roles where Nord intends them to, the contract is right; if it forces collisions, the spec is revised before further presets are added.

A patch milestone scoped to a single spec. Builds on {% ref "SPEC-048" /%} (token contract foundation) and the highlighter-agnostic `--rf-syntax-*` surface it established.

## Implementation order

1. **{% ref "WORK-217" /%}** — extend `SyntaxTokens` interface with the 7 new optional roles and JSDoc fallback documentation. Pure types change, no runtime impact.
2. **{% ref "WORK-218" /%}** — audit Shiki's stock css-variables theme to confirm which optional roles are already emitted vs. need wiring, then ship the extended css-variables theme in `@refrakt-md/highlight`.
3. **{% ref "WORK-219" /%}** — CSS variable generator emits `var()` fallback chains for each optional role.
4. **{% ref "WORK-220" /%}** — Nord preset module in `packages/lumina/src/presets/nord.ts` as the validation case.
5. **{% ref "WORK-221" /%}** — Nord preset documentation page with `{% palette %}` blocks honouring Nord's original swatch names.
6. **{% ref "WORK-222" /%}** — Cross-reference SPEC-056 from SPEC-048 so the foundation spec points to its syntax tier extension.

WORK-217 and WORK-218 can run in parallel. WORK-219 depends on WORK-217. WORK-220 depends on WORK-217 + WORK-219. WORK-221 depends on WORK-220. WORK-222 is independent.
