# v0.23.0 — Surface axes & the skeleton/skin extraction

The second slice of the {% ref "SPEC-094" /%} theme-platform epic. Where v0.22.0 tokenized
type and built the gallery safety net, v0.23.0 attacks the **differentiation surface**: it
extracts skeleton from skin and replaces the last cross-rune rune-name lists with
engine-emitted, theme-overridable config.

Its centerpiece is {% ref "SPEC-107" /%} — the **surface-axis decomposition**. It amends
{% ref "SPEC-094" /%} §8: the single `surface` enum (`card/banner/inline/inset`) splits into
three orthogonal, theme-defaulted axes — an ordered **`elevation`** depth ladder
(`sunken/flush/flat/raised/floating`, folding in today's shadow-only elevation), **`width`**
carrying the bleed, and a page-section-family **`prominence`** scale
(`quiet/normal/prominent/display`). The motivating outcome: the same `recipe` reads as a
bordered card in prose and a full-bleed, large-title hero at the top of a page.

## Surface axes ({% ref "SPEC-107" /%})

- {% ref "WORK-423" /%} — `elevation` depth-ladder axis: engine emission, value set, `defaultElevation` config, deprecation aliases off `none/sm/md/lg`. **The gate.**
- {% ref "WORK-424" /%} — `prominence` axis: engine emission, page-section-family availability gating, `defaultProminence` config.
- {% ref "WORK-425" /%} — Lumina mapping: rungs → chrome, prominence → type register, per-rune defaults; retire the static `surfaces.css` rune-name lists.
- {% ref "WORK-426" /%} — migration + docs: content codemod for the `elevation` rename, `surfaces.md` rewrite, migrate `figure`/`card`/`bento` + theme-authoring, changeset.
- {% ref "WORK-427" /%} — demonstration: full-bleed hero `recipe`/`playlist` examples (the original use case) in the gallery + docs.

## Skeleton/skin extraction

The wholesale extraction the v0.22.0 spike ({% ref "WORK-410" /%}) sized — packaging the
dimension + surface layers as a shared skeleton so theme #2 is a token file + layout +
targeted skin, not a fork. The surface-axis work above is the engine-side half it depends on.

- {% ref "WORK-428" /%} — spacing token contract + Lumina refactor. The prerequisite the spike surfaced (the spacing analog of WORK-405). **The gate.**
- {% ref "WORK-429" /%} — the `@refrakt-md/skeleton` package + `@layer skeleton, skin` infrastructure + loader order-emit.
- {% ref "WORK-430" /%} — icon-from-config: lift the embedded `data:image/svg+xml` glyphs out of CSS into the icon registry.
- {% ref "WORK-431" /%} — the wholesale skeleton/skin re-bucketing of Lumina's CSS (rune → dimension → layout), per the spike's cut-line rule.

## Sequencing

{% ref "WORK-410" /%} (v0.22.0 spike) set the cut line first. Two threads then run largely in
parallel:

- **Surface axes:** {% ref "WORK-423" /%} → {% ref "WORK-424" /%} → {% ref "WORK-425" /%} → {% ref "WORK-426" /%} → {% ref "WORK-427" /%}.
- **Skeleton/skin extraction:** {% ref "WORK-428" /%} (spacing tokens — the gate) → {% ref "WORK-429" /%} (skeleton package + layer infra) → {% ref "WORK-431" /%} (re-bucketing). {% ref "WORK-430" /%} (icons) lands alongside {% ref "WORK-423" /%}.

They converge in {% ref "WORK-431" /%}, which composes the surface treatment ({% ref "WORK-423" /%}) and icons ({% ref "WORK-430" /%}) into the skin layer.
