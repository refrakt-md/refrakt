# v0.15.0 — Composable embedding

The v0.15.0 milestone delivers the cluster of runes and infrastructure that makes refrakt's "embed anything inline" story real: a body-only drawer rune with xref-as-trigger ({% ref "SPEC-060" /%}), an `expand` rune that inlines registered entities ({% ref "SPEC-066" /%}), a `snippet` rune that embeds project files as syntax-highlighted code ({% ref "SPEC-062" /%}), and the supporting plumbing that ties them together: `$page.*`/`$file.*` page variables ({% ref "SPEC-061" /%}), generic file roots for namespaced file resolution ({% ref "SPEC-063" /%}), unconditional plan-entity registration ({% ref "SPEC-064" /%}), and configurable xref resolution with patterns and decoupled entity/URL lookup ({% ref "SPEC-065" /%}).

Once these land, the same `.md` file can render as its own canonical page, embed inline via expand, open as a drawer panel from a sentence in prose, or quote-link via xref — all from one source. That's the unique authoring story behind the future "see it in action" introduction article ({% ref "SPEC-067" /%}, deferred).

**Phase 2** — Plan content as entities, snippet rune:

**Phase 3** — Drawer:

**Phase 4** — Expand:
