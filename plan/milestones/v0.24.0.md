# v0.24.0 — Live backdrops & scroll-reveal motion

Two independent feature threads, split out of v0.23.0 so the theme-platform extraction stays
focused and these ship on their own cadence. Neither depends on the v0.23.0 surface/skeleton
work; each is gated only on its own foundation.

## `bg` live sandbox guests ({% ref "SPEC-104" /%})

Let `bg` host a live `sandbox` as a bare, full-bleed backdrop — the layer that lets a
surface carry both an animated backdrop and a positioned subject media.

- {% ref "WORK-428" /%} — guest body + engine relocation + bare-surface guardrail (the `data-bg-guest` contract). **The gate.**
- {% ref "WORK-429" /%} — transform-resolved `sandbox` bg preset (`BgPresetDefinition.sandbox`, schema, project-config home).
- {% ref "WORK-430" /%} — docs + music-blog backdrop showcase (layout reuse + {% ref "SPEC-006" /%} bridge cross-link).

## Scroll-reveal motion ({% ref "SPEC-105" /%})

A token-driven motion dimension: a closed `reveal` vocabulary + `stagger`, author intent /
theme choreography / IO-triggered, enhancement-gated.

- {% ref "WORK-431" /%} — `reveal`/`stagger` engine facet (`data-reveal`, `--rf-reveal-index`). **The gate.**
- {% ref "WORK-432" /%} — motion dimension CSS + tokens (global hooks; individual `translate`/`scale` composition).
- {% ref "WORK-433" /%} — `reveal` IntersectionObserver behaviour + enhancement gating.
- {% ref "WORK-434" /%} — docs + feature-stagger showcase.
