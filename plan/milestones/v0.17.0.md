# v0.17.0 — Declarative metadata & layout

The v0.17.0 milestone follows v0.16.0's registry-driven sites with a fully
declarative model for how metadata-bearing and media-bearing runes are
assembled — replacing per-rune imperative structure code with a small,
orthogonal config vocabulary (`metaFields` + `blocks` + `layout`) and giving
every rune a consistent, theme-overridable metadata treatment. It also lands
the `file-ref` rune and the shared `preview` drawer.

## Goals

- A fully declarative metadata-and-layout model for rune assembly
  ({% ref "SPEC-080" /%}, building on {% ref "SPEC-079" /%}): every meta-bearing
  rune projects its metadata via `metaFields` + `blocks` + `layout`, with new
  `bar` / `definition-list` primitives, the `{% bar %}` / `{% deflist %}`
  authoring runes, a universal metadata chip system, and rich field renderings
  (link / rating / icon). This is the headline of the release.
- Land the `file-ref` rune + shared `preview="…"` attribute on `xref`, with
  single-block drawer styling, per {% ref "SPEC-078" /%}.

Deferred to v0.18.0: the html-scaffold client-behaviors bundle
({% ref "WORK-293" /%}) and the `plan-progress`-over-`aggregate` decomposition
({% ref "WORK-296" /%}). The legacy `slots` + `structure` removal is also
v0.18.0 ({% ref "WORK-313" /%}), so v0.17.0's model is purely additive.
