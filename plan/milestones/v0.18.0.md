# v0.18.0 — Contract hardening

Follows v0.17.0's declarative metadata-and-layout model. The release hardens
the rune output contract before third-party themes build against it, and lands
the one breaking change held back from v0.17.0.

## Goals

- Remove the legacy `slots` + `structure` shim now that every first-party rune
  is on `metaFields` + `blocks` + `layout` ({% ref "WORK-313" /%}) — the one
  breaking change held back from v0.17.0, with its own changeset.
- Land the rune-contract-hardening specs end to end: declarative structure
  assembly (`SPEC-081`) and the typed node data channel (`SPEC-082`).

The html-scaffold client-behaviors bundle ({% ref "WORK-293" /%}) and the
`plan-progress` aggregate refactor ({% ref "WORK-296" /%}) were deferred to
{% ref "v0.19.0" /%} to keep this release focused on the contract surface.
