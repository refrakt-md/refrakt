# v0.29.0 — Multi-language support (i18n foundation)

Makes Refrakt's framework-generated text — labels, navigation chrome, accessibility strings,
structural headings, and client-side behavior strings — localizable, so a site built for a
non-English locale no longer shows English UI chrome around translated content. This milestone
delivers the **string-resolution foundation** of {% ref "SPEC-035" /%}. Serving the same site in
multiple languages (route trees, switcher, auto-detect) remains a deliberate Non-Goal that this
foundation is explicitly constrained to allow later without breaking changes.

## Shape

Every localizable string resolves through one path — a locale-selected `strings` dictionary on
`ThemeConfig`, keyed by **auto-derived** `{scope}.{block}.{ref}` keys, with English as the
zero-config fallback. Server-side text resolves through a narrow `LocaleContext` slice threaded into
the engine; client-side behavior strings are delivered **inline** (`data-i18n-*` attributes + a
per-page JSON block), never fetched. Plugins ship their own per-locale JSON bundles, and refrakt
ships first-party bundles for popular languages. See the {% ref "SPEC-035" /%} Decisions section for
the eight resolved design questions (keys, plurals, format, delivery, fallback, threading,
first-party, packaging).

Foundation first, then the zones fan out:

- **Foundation (P0):** {% ref "WORK-502" /%} (locale/strings/`LocaleContext`/resolvers/fallback) →
  {% ref "WORK-503" /%} (thread `LocaleContext` into the engine label renderers).
- **Core surfaces (P1):** {% ref "WORK-504" /%} (structure & meta-field labels + auto-derived keys),
  {% ref "WORK-505" /%} (layout chrome + computed transforms), {% ref "WORK-506" /%}
  (`refrakt i18n extract` + `--check` + MCP).
- **Secondary surfaces (P2):** {% ref "WORK-507" /%} (behavior strings, inline delivery),
  {% ref "WORK-508" /%} (postTransform / plan-render + enum-as-text), {% ref "WORK-509" /%}
  (plugin translation bundles + `mergePlugins` wiring).
- **Follow-ups (P3):** {% ref "WORK-510" /%} (knownSections `canonicalSlug` + `i18nAliases` — the
  stable-slug follow-up; can start in parallel since knownSections already shipped),
  {% ref "WORK-511" /%} (cross-adapter `lang` + Intl number/duration/currency),
  {% ref "WORK-512" /%} (first-party translation bundles — phased, demand-driven).

Scope note: content translation, RTL layout, and CLI-string i18n are Non-Goals ({% ref "SPEC-035" /%}).
